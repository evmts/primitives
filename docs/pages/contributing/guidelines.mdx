# Code Guidelines

## Core Principles

- **No stubs** - Never use `error.NotImplemented` or placeholder implementations
- **Test everything** - Write tests alongside code
- **Memory safety** - Explicit ownership and cleanup
- **Performance** - Optimize hot paths

## Code Style

- Use `snake_case` for variables
- Use `PascalCase` for types
- Use `camelCase` for functions
- Format with `zig fmt`

See [CLAUDE.md](https://github.com/williamcory/Guillotine/blob/main/CLAUDE.md) for detailed guidelines.
